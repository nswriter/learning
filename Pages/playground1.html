Changing the Image

This page utilizes an Event Listener Javascript tag to "listen" for a click on the page. This variable, named myImage, is tied to a document.querySelector("img") which means it will select all img tags on the page, and thus only accept clicks within the content of an img tag.

When a click is registered (or I suppose heard) it uses an if/else statement to modify the value of a variable myImage. This variable is, by default, set to the image "Ship1", the arid landscape. When clicked, the variable changes the path using the if statement to lead to the "Ship2" image source, thus changing the displayed image to the arctic landscape. If clicked again, the variable uses the "else" statment to reset the image source to the "Ship1" path, since it has detected that the image is not on "Ship1" already, and was thus changed, and so it needs to change it back. Each click will continue to cycle back and forth.

If there were other images being used on this page, then it would be wise to replace the general ("img") selector with a more specific selector using a class name or id, something to tie the script to that specific image. Otherwise, all images on the page, regardless of their content, would suddenly flip irrevocably to "Ship1", because the "else" statement would still detect a lack of a "Ship1" source and automatically fill it back in, overwriting whatever source the other image had, and then locking this image into the same back and forth cycle between 1 and 2, never returning to the original image (until the page were to be refreshed, of course).

Note: I also added another constant using a class-based selector to change the caption for the images! I want to be able to change the alt text, too, but that might just be a task for another day.

Changing the User Name Welcome Message

I must admit I found this little trick quite fun to mess around with. While in this basic state it lacks any true weight behind the "User" (that is only the name data item is stored, singularly, and only until it is overwritten--there is no retreivable profile) it grants the illusion, while providing an interesting building block for later experiments.

This function uses a function (setUserName) to create a data item through the Web Storage API, meaning that whatever value gets assigned to it will be retained through localStorage on the web browser, retaining the information even if the browser gets closed. This means that, at least for a longwhile, this page will remember whatever name you entered as your username. Pretty neat! More reaserch is required.

The functionality of this function is very similar in principle to the image selector above, but with some notable differences to account for its context. The function uses another querySelector, this time set to a class of ".Username", so that it interacts only with the one "p" tag in the header where the message is to be displayed. It would be pretty awkward if all of these notes just turned into a long list of "Welcome, y/n". I must say, the original tutorial didn't have notes, or titles, or anything of that sort, so I leveraged my past knowlege of HTML and CSS selector interactions, and indeed it works much the same. So that's cool.

The other difference to the image selector is the functionality surrounding the popup. To summon this popup, and accept a text input, this function utilizes a nifty feature called prompt which accepts a string, which will be the text displayed in the popup. It also summons the popup itself, with the neccessary functionality built in. There is an issue, by default, where you can choose to click "OK" without having entered a name, in which it would display "Welcome,", or you could click "Cancel" and it would close the popup and display "Welcome, null", with "null" being the default reply for an empty variable in Javascript. To cicrumvent this for the moment, the tutorial includes what is esentially a copy of the myName constant, within the same setUserName function, but this time includes an intervening "logical NOT" gate, (!localStorage.getItem("name")) to indicate that if localStorage does NOT have a name data item stored, the next if/else statement should be run. This if/else statement is, as noted, quite similar to the previous except that it is calling upon const storedName instead of const myName. This means that unless an input is added, creating or overwriting the "name" data unit, it will continue to relaunch the popup. Once a "name" data unit is found, it becomes the storedName, and the else half of the function will write that name into the ".Username" designated "p" tag, and all is well.

The "Change User" button simply uses another querySelector to detect if the button is pressed, and if it is, it launches the setUserName function (I spelled that funtion so many times) above, allowing the stored user name to be, well, changed.
